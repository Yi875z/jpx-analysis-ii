## 2. 明日以降やること: n8n自動実行の設定

### n8nへのインポート手順

1. ブラウザで `http://localhost:5678` を開く
2. 左サイドバー「Workflows」→「New」→「Import from file」
3. `C:\CarSol\jpx-analysis\config\n8n_workflow.json` をインポート
4. Gmailクレデンシャルを設定する（下記）
5. 「Active」トグルをONにする

### Gmailクレデンシャルの設定

n8nでGmail送信するには Google OAuth2 の設定が必要:

1. n8n の「Credentials」→「Add Credential」→「Gmail OAuth2」
2. GoogleのOAuth2クライアントID・シークレットを入力
   - 取得先: Google Cloud Console → APIとサービス → 認証情報
   - リダイレクトURI: `http://localhost:5678/rest/oauth2-credential/callback`
3. n8n の `NOTIFY_EMAIL` 環境変数に通知先メールアドレスを設定

**もしGmailが面倒な場合の代替:** 完了メールの代わりに n8n の「Webhook」ノードでSlackやLINEに通知する方法も簡単。

### Supabase無料版の制約と対策

| 制約 | 内容 | 対策 |
|---|---|---|
| **自動停止** | 7日間APIアクセスなしでプロジェクトが停止 | n8nワークフローの最初にウェイクアップリクエストを追加済み |
| ストレージ | 500MB | 何年分でも余裕。年間約1MB未満 |
| プロジェクト数 | 2個まで | このプロジェクト1個で完結 |

**停止した場合の手動復旧手順:**
1. `https://supabase.com/dashboard` を開く
2. プロジェクト `syyojlcrnachuvrbvttw` を選択
3. 「Restore project」ボタンをクリック（復旧に1〜2分）
4. `python main.py` を手動実行
