## 発見・修正した問題

### 問題① 【最重大】投資家コードの誤マッピング

`scripts/parse_futures_csv.py` の `INVESTOR_CODES` でコード30と50が完全に逆だった。
PDFの実測値と照合して全コードを検証・追加。

**修正前（誤）:**
```python
INVESTOR_CODES = {
    60: "foreign",
    30: "individual",        # 誤: コード30は法人計
    50: "institution_total", # 誤: コード50は個人計
    23: "trust_bank",
    32: "corporate",
    10: "dealer",
}
```

**修正後（正・v2.1）:**
```python
INVESTOR_CODES = {
    10: "dealer",             # 自己（証券会社）
    20: "financial_total",    # 金融機関計
    21: "insurance",          # 生保損保
    22: "city_regional_bank", # 都銀地銀
    23: "trust_bank",         # 信託銀行
    24: "other_financial",    # その他金融機関
    30: "institution_total",  # 法人計
    31: "investment_trust",   # 投資信託
    32: "corporate",          # 事業法人
    33: "other_institution",  # その他法人
    41: "securities",         # 証券会社
    50: "individual",         # 個人計
    60: "foreign",            # 海外投資家計
}
```

影響: 旧レポートでは「個人投資家 +6,617億円（買越）」と誤表示。
正しくは「法人計 +6,618億円（買越）」「個人計 -1,413億円（売越）」で方向性が逆だった。

検証方法: `Tousi_DV_W_202603_4_0323_0327.pdf` 1ページ目（日経225先物）の投資家別表と
CSV col7の各コード枚数（法人計コード30: 差引+3,342、個人計コード50: 差引-710）を照合。

### 問題② 集計商品の範囲（日経225マイクロを除外）

サイトが集計する商品は4商品。日経225マイクロ（コード331）は対象外。

| 商品コード | 商品名 | 単位 | 乗数 |
|-----------|--------|------|------|
| 301 | 日経225ラージ | 千円 | 1,000 |
| 313 | 日経225mini | 円 | 1 |
| 314 | TOPIXラージ | 百円 | 100 |
| 316 | TOPIXmini | 円 | 1 |
| ~~331~~ | ~~日経225マイクロ~~ | ~~除外~~ | - |

検証: 4商品で計算すると全投資家区分でサイト数値と±1億円以内に一致。

修正後の `FUTURES_CODES`:
```python
FUTURES_CODES = {
    301: ("nikkei225_large", 1_000),
    313: ("nikkei225_mini",  1),
    314: ("topix_large",     100),
    316: ("topix_mini",      1),
    # 331: ("nikkei225_micro", 1),  # サイト集計対象外のため除外
}
```

## 明日以降の作業指示

### 【優先度 高】次週データの取り込み

JPXは毎週木曜に翌週データを公開。2026-04-03週のファイルをダウンロードして実行。

ダウンロード先: https://www.jpx.co.jp/markets/statistics-derivatives/sector/

```powershell
cd C:\CarSol\jpx-analysis

# Step1: 先物CSV保存（ファイル名と日付を差し替える）
python save_futures_to_db.py Tousi_DV_W_202604_1_0330_0403.csv 2026-04-03

# Step2: 現物XLS保存
python save_to_db.py stock_val_1_260403.xls 2026-04-03

# Step3: レポート再生成
python main.py --report-only --date 2026-04-03
```

### 【優先度 中】現物データの集計範囲を東京・名古屋に統一

`scripts/parse_spot_xls.py` が読み込むシートを「東証プライム」から「東京・名古屋市場」に変更すると
サイトとの約4億円の差が解消される。ファイルを読んで対象シート名を確認してから修正する。

```powershell

python save_futures_to_db.py <CSV名> <YYYY-MM-DD>
```
